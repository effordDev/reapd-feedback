# reapd — Feedback & Roadmap

This is the public issue tracker and roadmap for [reapd](https://reapd-production.up.railway.app) — a web app for deploying SFDX GitHub repositories to Salesforce orgs.

---

## Report a Bug

Found something broken? [Open a bug report](../../issues/new?template=bug_report.md) and include:
- What you were trying to do
- What happened vs. what you expected
- Any error messages you saw

---

## Request a Feature

Have an idea? [Open a feature request](../../issues/new?template=feature_request.md) and tell us what problem it solves.

---

## Roadmap

These are features we're actively thinking about. Community votes and feedback help prioritize what gets built next.

### In Progress
- Custom domain support

### Planned
- **Changelog** — in-app release notes so you always know what's new
- **Deploy notifications** — email when a long deployment finishes
- **Usage limits** — free tier deploy caps to encourage sustainable growth
- **Status page** — public uptime page so you can check if reapd is having issues

### Ideas — Help Us Prioritize
Got a use case for any of these? Comment on the idea or open a new one!

- **Multi-user / team support** — share Salesforce org connections across a team
- **Scheduled deploys** — trigger a deployment at a set time
- **Slack / Teams notifications** — post deploy results to a channel
- **Deploy rollback** — re-deploy a previous commit SHA
- **Org health check** — quick validation of your Salesforce org before deploying
